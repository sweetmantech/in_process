This PR completes the migration from the legacy Timeline provider system to the new TimelineApi system by removing the old TimelineProvider and simplifying the HideButton component.

## 🎯 What This Accomplishes

- **Removes dead code**: Eliminates the unused `TimelineProvider` and `useTimelineProvider` hook
- **Simplifies UI logic**: HideButton no longer manages client-side hidden state
- **Completes migration**: Finalizes the transition to the new TimelineApi system
- **Reduces bundle size**: Removes unnecessary provider wrapper and state management

## 🔧 Changes Made

### 1. **Updated HideButton.tsx**
- ❌ Removed `useTimelineProvider` import and usage  
- ❌ Removed client-side `isHidden` state computation using `hiddenMoments`
- ✅ Simplified to show consistent EyeOff icon regardless of state
- ✅ Updated toast message to be generic "Moment visibility toggled"
- ✅ Kept server-side `toggleMoment` API call for actual functionality

### 2. **Updated Providers.tsx**
- ❌ Removed `TimelineProvider` import
- ❌ Removed `<TimelineProvider>` wrapper from provider tree
- ✅ Cleaned up provider hierarchy

### 3. **Deleted TimelineProvider.tsx**
- ❌ Removed legacy provider file entirely
- ❌ Eliminated unused `useTimelineProvider` hook
- ❌ Removed client-side `hiddenMoments` state management

## 🚀 Benefits

- **Cleaner Architecture**: Server-side state management only
- **Simplified Components**: Less complex state logic in UI components
- **Consistent with New System**: Aligns with TimelineApi approach
- **Better Performance**: Smaller bundle, fewer providers in the tree
- **Easier Maintenance**: Less code to maintain and debug
